# Как интерпретировать FINAL_REPORT.md

Этот документ описывает каждый раздел итогового отчёта `FINAL_REPORT.md`,
объясняет значение всех метрик и даёт ориентиры для оценки качества результатов.

---

## Общая структура отчёта

| Раздел | Что проверяет |
|--------|--------------|
| Experiment Status | Все ли эксперименты завершились без ошибок |
| 1. Training Results | Качество обучения нейросети |
| 2. Band Prediction Accuracy | Точность предсказания полосы (band) |
| 3. Pairwise Speedup | Ускорение парного выравнивания vs Full NW |
| 4. Ablation Study | Вклад каждого компонента (Fixed vs Neural) |
| 5. Four Russians | Эффективность таблицы Four Russians |
| 6. MSA Quality | Качество множественного выравнивания |
| 7. Scaling by N | Масштабируемость по числу последовательностей |
| 8. Test Results | Прохождение автотестов |
| 9. System Information | Конфигурация оборудования |

---

## Experiment Status (Таблица в начале)

Показывает статус каждого эксперимента (`OK` / `ERROR`) и время выполнения.

- **Все `OK`** — ожидаемое поведение. Если есть `ERROR`, см. файлы
  `results/experiments/<name>_ERROR.json` для деталей.
- **Время** — полезно для оценки вычислительных затрат. MSA quality обычно
  занимает больше всего (10–30 минут на 30 групп ДНК).

## 1. Neural Network Training Results

### Best Epoch Metrics

| Метрика | Описание | Хорошо | Плохо |
|---------|----------|--------|-------|
| **band_recall@1.0x** | Доля пар, для которых предсказанная ширина полосы ≥ истинной. Выравнивание пройдёт без удвоения. | ≥ 0.80 | < 0.50 |
| **band_recall@1.5x** | То же, но с запасом 1.5×. Т.е. `pred_hw × 1.5 ≥ true_hw`. | ≥ 0.90 | < 0.70 |
| **band_recall@2.0x** | С запасом 2×. Практически все пары покрыты. | ≥ 0.95 | < 0.85 |
| **MAE centre_diag** | Средняя абсолютная ошибка предсказания центральной диагонали (в позициях). | < 10 | > 50 |
| **width_ratio** | Отношение `pred_hw / true_hw`. Значение > 1 = сеть слегка завышает ширину (безопасно). Значение < 1 = сеть занижает (опасно, неправильные выравнивания). | 1.0–2.0 | < 0.8 или > 5.0 |

### Training History

Таблица по эпохам. Ожидается:
- **Loss** монотонно снижается.
- **Recall@1x** растёт от эпохи к эпохе.
- Если loss не снижается после 2–3 эпох — модель не обучается (проблема
  с данными или learning rate).

## 2. Band Prediction Accuracy

Оценка точности предсказания на валидационной выборке, разбитой по уровням дивергенции.

### Группы дивергенции

| Группа | Диапазон | Что означает |
|--------|----------|-------------|
| **low** | 0–10% | Очень похожие последовательности. Полоса узкая, предсказать легко. |
| **medium** | 10–25% | Средняя дивергенция. Основной рабочий диапазон. |
| **high** | 25–50% | Сильно расходящиеся. Полоса широкая, предсказать сложнее. |

### Как читать

- **Recall@1x по группам** — главная метрика. Показывает, как часто сеть
  предсказывает достаточно широкую полосу с первого раза.
  - `low ≈ 0.95+` — ожидаемо, здесь полосы узкие и регулярные
  - `medium ≈ 0.80+` — хороший результат
  - `high ≈ 0.60+` — приемлемо, для сложных пар допустимы удвоения
- **MAE centre** — чем ниже, тем лучше. Для `low` ожидается 0–5, для `high`
  допустимо 10–30.
- Если `recall@1x` для `high` группы < 0.5 — модель не справляется с дальними
  эволюционными расстояниями (normal для ранней стадии обучения).

## 3. Pairwise Alignment Speedup

Сравнение времени выравнивания одной пары последовательностей:
- **Full NW** — полный алгоритм Нидлмана-Вунша (O(n²)), baseline
- **Fixed W=30** — фиксированная полоса шириной 30
- **Fixed W=100** — фиксированная полоса шириной 100
- **Neural band** — наш метод (нейросеть предсказывает ширину)

### Ключевые числа

| Метрика | Описание | Хорошо | Ожидаемо |
|---------|----------|--------|----------|
| **Mean speedup vs Full NW** | Во сколько раз быстрее полного NW | > 5× | 3–20× |
| **Max speedup** | Максимальное ускорение (на длинных последовательностях) | > 20× | 10–100× |
| **Speedup vs Fixed W=30** | Сравнение с фиксированной полосой. Если > 1, наша полоса уже. Если < 1, наша шире (но точнее). | 0.5–2× | ~1×|

### Detailed Table

- Ускорение растёт с длиной последовательности — это главный аргумент метода.
- На коротких (300 bp) ускорение может быть < 2× — overhead нейросети заметен.
- На длинных (5000 bp) ожидается 10–50× vs Full NW.
- При высокой дивергенции (30%) полоса шире → ускорение меньше. Это нормально.

## 4. Ablation Study

Показывает вклад каждого компонента: какая часть ускорения приходится на
banded-подход, а какая — на нейросетевое предсказание.

### Как читать таблицу

| Метод | Интерпретация |
|-------|--------------|
| **Full NW** | Baseline, speedup = 1.0× |
| **Fixed W=50** | Ускорение только от banded-подхода с фиксированной шириной |
| **Fixed W=100** | Более широкая полоса — медленнее, но надёжнее |
| **Neural band** | Наш метод. Ширина подбирается под каждую пару |

### Ключевые метрики

- **mean_doublings** — среднее число удвоений полосы. Идеал = 0. Если > 1, сеть
  часто предсказывает слишком узкую полосу, и алгоритм вынужден расширять.
- **zero_doubling_rate** — доля пар, где удвоения не потребовались. Хорошо: > 0.8.
  Означает, что нейросеть сразу подобрала правильную ширину.

### Интерпретация

- Если Neural band быстрее Fixed W=50, но медленнее Fixed W=100 — значит
  нейросеть предсказывает ширину между 50 и 100, адаптивно.
- Если doublings > 0.5 — модель недообучена или данные слишком сложные.
- Speedup Neural vs Full NW — должен быть сопоставим с Fixed W=50, но с
  гарантией корректности (recall ≈ 1.0 при достаточном числе удвоений).

## 5. Four Russians Lookup Table Accumulation

Эксперимент показывает, как hit_ratio (доля повторного использования
предвычисленных блоков) растёт по мере обработки пар.

### Ключевые метрики

| Метрика | Описание | Хорошо |
|---------|----------|--------|
| **Initial hit_ratio** | После первых 10 пар | 0.0–0.1 (таблица пустая) |
| **Final hit_ratio** | После 300 пар | > 0.5 (половина блоков из кэша) |
| **Table size** | Размер таблицы в KB | < 10 МБ |

### Интерпретация

- Hit_ratio растёт логарифмически — быстро вначале, потом выходит на плато.
- Высокий hit_ratio (> 70%) означает, что Four Russians значительно ускоряет
  внутреннюю DP-процедуру при banded-выравнивании.
- Если hit_ratio не растёт — блоки `t × t` слишком велики или последовательности
  слишком разнородные.
- Размер таблицы должен быть << доступной RAM.

## 6. MSA Quality

**Самый важный раздел для диссертации.** Сравнение нашего метода с классическими
MSA-инструментами на синтетических ДНК-данных с известным эталонным выравниванием.

### Методы

| Метод | Категория | Описание |
|-------|-----------|----------|
| **ClustalW** | Классический | Прогрессивный MSA, устаревший стандарт |
| **MAFFT** | Классический | Быстрый и точный, текущий gold standard |
| **MUSCLE** | Классический | Итеративный MSA, хорошее качество |
| **Fixed_W30** | Ablation | Наш progressive MSA с фиксированной полосой W=30 |
| **Fixed_W100** | Ablation | Наш progressive MSA с фиксированной полосой W=100 |
| **Neural_band** | Наш метод | Progressive MSA с нейросетевой полосой |
| **Neural_+_refine** | Наш метод + | Neural_band + итеративное уточнение |

### Метрики качества

| Метрика | Описание | Диапазон |
|---------|----------|----------|
| **SP-score** (Sum-of-Pairs) | Доля правильно выровненных пар остатков. Сравнивает каждую пару позиций в предсказанном выравнивании с эталонным. | 0.0–1.0 |
| **TC-score** (Total Column) | Доля полностью правильных столбцов. Более строгая метрика — столбец засчитывается только если ВСЕ остатки совпадают с эталоном. | 0.0–1.0 |
| **Time (s)** | Среднее время на одну группу | зависит от N и длин |
| **Memory (MB)** | Пиковое потребление RAM на одну группу | зависит от N и длин |

### Ориентиры для SP-score

| SP-score | Интерпретация |
|----------|--------------|
| > 0.8 | Отличное выравнивание, на уровне MAFFT |
| 0.5–0.8 | Приемлемое, есть ошибки но основная структура верна |
| 0.2–0.5 | Слабое, значительная часть пар выровнена неправильно |
| < 0.2 | Очень плохое, выравнивание практически случайное |

### Как оценивать наш метод

1. **Neural_band vs MAFFT** — главное сравнение. SP-score нашего метода
   **не должен быть намного ниже** MAFFT (допустимо на 10–30% ниже, т.к.
   наш метод оптимизирует скорость, а не только качество).
2. **Neural_band vs Fixed_W30/W100** — показывает преимущество адаптивной
   ширины. Neural_band должен быть ≥ Fixed_W100 по SP-score.
3. **Neural_+_refine vs Neural_band** — итеративное уточнение должно слегка
   улучшать SP/TC (на 0.02–0.10).
4. **Time** — наш метод должен быть быстрее MAFFT при большом N, или
   сопоставим при малом N.
5. **SP = -1.0** в деталях означает, что для этой группы не удалось сопоставить
   предсказанные и эталонные последовательности (ошибка в данных, не в методе).

## 7. Scaling by Number of Sequences N

Показывает, как время работы зависит от числа последовательностей (N = 10..100).

### Как читать

| N | Ожидаемое поведение |
|---|---------------------|
| 10 | MAFFT и Neural примерно одинаковы |
| 20 | Neural может быть чуть медленнее (overhead нейросети) |
| 50 | Neural должен быть быстрее или сопоставим |
| 100 | При большом N преимущество banded-подхода растёт |

### Интерпретация

- **Speedup > 1** — наш метод быстрее MAFFT. Ожидается при N ≥ 50.
- **Speedup < 1** — MAFFT быстрее. Допустимо при малом N (MAFFT высоко
  оптимизирован).
- Если speedup не растёт с N — bottleneck не в парных выравниваниях, а в
  guide tree или конкатенации профилей.

## Типичные проблемы и их диагностика

| Симптом | Вероятная причина | Что делать |
|---------|-------------------|------------|
| Все SP = 0.0 | Последовательности не сопоставились между pred и ref | Проверить порядок seq_ids; см. диагностику в run_all.py |
| SP = -1.0 | Нет валидного эталона для этой группы | Проверить данные; группа может не иметь aligned reference |
| recall@1x < 0.5 | Модель сильно недообучена | Увеличить число эпох, проверить данные |
| Speedup < 1× | Overhead нейросети больше выигрыша | Нормально для коротких последовательностей (< 500 bp) |
| hit_ratio = 0 | Four Russians таблица не заполняется | Проверить параметр t (размер блока) |
| Time = 999s | Метод упал с ошибкой | Смотреть error в JSON-деталях |

   правильную ширину полосы.
2. **Mean speedup vs Full NW** из раздела 3 — главный количественный результат
