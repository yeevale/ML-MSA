## 1. Общие сведения о работе

**Тема:** Нейросетевое предсказание диагональной полосы для ускорения
множественного выравнивания последовательностей

**Область:** Биоинформатика / Машинное обучение

**Суть в одном абзаце:**
Множественное выравнивание последовательностей (MSA) — одна из фундаментальных
задач биоинформатики. Классический алгоритм Нидлмана-Вунша (NW) для попарного
выравнивания имеет сложность O(n²), что становится узким местом при обработке
тысяч последовательностей. Мы предлагаем использовать обученную нейронную сеть
для предсказания оптимальной ширины и центра диагональной полосы (band) в
матрице динамического программирования, сокращая вычисления до O(n·W), где
W << n. Ключевое свойство: корректность гарантирована механизмом удвоения
полосы (band doubling) — если оптимальный путь выходит за пределы предсказанной
полосы, она автоматически расширяется.

## 2. Постановка задачи

### 2.1. Что такое MSA

**Множественное выравнивание последовательностей** — задача расположения N
биологических последовательностей (ДНК или белков) друг под другом с вставкой
пропусков (gaps, символ `-`) так, чтобы гомологичные позиции оказались в одном
столбце. Результат — прямоугольная матрица символов одинаковой ширины.

**Зачем:** поиск консервативных мотивов, построение филогенетических деревьев,
предсказание структуры белков, эволюционный анализ.

**Пример:**
```
seq1: ACGT--ACGT
seq2: AC--GTACGT
seq3: --ACGTACGT
```

### 2.2. Что такое banded NW

Алгоритм Нидлмана-Вунша строит матрицу размером L₁ × L₂ и заполняет каждую
ячейку. При больших длинах (L > 1000) это дорого.

**Наблюдение:** оптимальный путь выравнивания почти всегда лежит вблизи
диагонали. Если заранее знать, где именно, — можно считать только узкую
полосу шириной 2W+1 вокруг предсказанного центра.

```
     Полная матрица NW          Banded NW (W=3)
  ┌─────────────────┐       ┌─────────────────┐
  │ ############### │       │     ###         │
  │ ############### │  →    │      ###        │
  │ ############### │       │       ###       │
  │ ############### │       │        ###      │
  └─────────────────┘       └─────────────────┘
    O(n²) ячеек               O(n·W) ячеек
```

**Сложность:** O(n·W) вместо O(n²). При W=50 и n=5000 → ускорение в 50×.

### 2.3. Проблема выбора W

- **W слишком мал** → оптимальный путь выходит за полосу → неправильное
  выравнивание
- **W слишком велик** → теряется смысл, приближаемся к O(n²)
- **W фиксирован** → не адаптируется к конкретной паре последовательностей

**Наше решение:** нейросеть предсказывает (centre, W) индивидуально для
каждой пары. Если предсказание неточное — band doubling гарантирует
корректность за счёт O(log(n/W)) дополнительных попыток.

## 3. Архитектура системы

### 3.1. Общая схема пайплайна

```
Входные последовательности (N штук)
         │
         ▼
┌─────────────────────────┐
│ 1. Матрица расстояний   │  k-mer Jaccard расстояния, O(N²)
│    между всеми парами   │  параллелизация через joblib
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│ 2. Guide tree           │  Neighbor-Joining или UPGMA
│    (направляющее дерево) │  scipy.cluster.hierarchy
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│ 3. Прогрессивное        │  Обход дерева снизу вверх (post-order)
│    выравнивание         │  На каждом внутреннем узле:
│                         │    a) Нейросеть → (centre, W)
│                         │    b) C++ banded NW → выравнивание
│                         │    c) Построение профиля
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│ 4. Итеративное          │  MUSCLE-style refinement
│    уточнение (опц.)     │  3 прохода, принять если SP лучше
└─────────┬───────────────┘
          │
          ▼
    Финальное MSA
```

### 3.2. Ключевые оптимизации

| Оптимизация | Где | Эффект |
|-------------|-----|--------|
| **Batched NN inference** | Progressive MSA | Узлы одного уровня дерева предсказываются одним батчем на GPU |
| **Lazy profiles** | Progressive MSA | Профили детей удаляются сразу после merge → O(log N) памяти |
| **Band doubling** | C++ aligner | Корректность при любом предсказании; асимметричное расширение |
| **Four Russians** | C++ aligner | Lookup-таблица t×t блоков DP → амортизированное ускорение |
| **SIMD (AVX2)** | C++ aligner | Векторизация антидиагональных вычислений |
| **Hirschberg** | C++ aligner | O(W) памяти вместо O(n·W) при обратном проходе |
| **Anchor mode** | Progressive MSA | Для последовательностей > 5000 bp: разбиение на блоки по якорям |
| **Feature caching** | Training | Кэширование .npz файлов с признаками → не пересчитывать |
| **torch.compile** | Inference | JIT-компиляция модели → +15–30% скорости на GPU |

## 4. Нейронная сеть

### 4.1. Архитектура модели

**Тип:** Гибридная CNN + MLP с эмбеддингом типа последовательности

```
                    Вход
                   /    |     \
    Similarity     Scalar     SeqType
     Matrix       Features    Embedding
    (1×64×64)      (70-dim)    (2→8)
        │             │           │
   ┌────┴────┐   ┌────┴────┐     │
   │ DotPlot │   │ Scalar  │     │
   │  CNN    │   │  MLP    │     │
   │ 4 блока │   │ 3 слоя  │     │
   │ Conv-   │   │ Linear- │     │
   │ BN-ReLU │   │ LN-ReLU │     │
   │ Pool    │   │ Dropout │     │
   └────┬────┘   └────┬────┘     │
        │(256)        │(64)    │(8)
        └──────┬──────┴────────┘
               │ concat (328)
         ┌─────┴─────┐
         │   Head    │
         │ 328→128   │
         │ →32→2     │
         └─────┬─────┘
               │
        (centre_diag,
         log_half_width)
```

### 4.2. Входные признаки (70 скалярных + матрица)

**Скалярные признаки (70-мерный вектор):**

| Группа | Признаки | Кол-во |
|--------|----------|--------|
| Длины | len₁/10000, len₂/10000, len₁/len₂, \|len₁−len₂\|/max | 4 |
| Состав | GC-content (ДНК) или charged fraction (белки) | 2 |
| K-mer статистики | Jaccard для k∈{3,4}, cosine sim, L1 dist | 6 |
| Минимайзеры | Jaccard минимайзеров (w=10, k=4) | 1–2 |
| Энтропия | Shannon entropy по k-mer частотам | 2 |
| Padding | Нули до размерности 70 | ~53 |

**Матрица сходства (1×64×64 тензор):**
1. Строим бинарный dot-plot: `[i,j] = 1` если k-mer на позиции i в seq₁
   совпадает с k-mer на позиции j в seq₂
2. Сжимаем до 64×64 билинейной интерполяцией (scipy.ndimage.zoom)
3. Min-max нормализация в [0, 1]

**Для профилей (внутренние узлы дерева):**
- Вместо dot-plot: матрица сходства профилей через BLOSUM62 / DNA subst
- Скалярные признаки: длина, энтропия, доля gap'ов, сходство сэмплированных позиций

### 4.3. Выход модели

| Выход | Формула | Смысл |
|-------|---------|-------|
| `centre_diag` | Линейный (без ограничений) | Смещение центра полосы от главной диагонали |
| `half_width` | exp(log_half_width) | Ширина полосы (всегда > 0 благодаря exp) |

### 4.4. Функция потерь

```
L = L_centre + λ · L_width

L_centre = MSE(pred_centre, true_centre)

L_width = AsymmetricHuber(pred_hw, true_hw, α=5.0)
```

**Asymmetric Huber:** штрафует недооценку ширины в α=5 раз сильнее, чем
переоценку. Это правильно: недооценка → band doubling (дорого), переоценка →
просто чуть больше вычислений.

### 4.5. Размер модели

~630 000 обучаемых параметров. Модель компактная, inference быстрый (< 1 мс
на batch из 32 пар на RTX 4090).

## 5. Обучение

### 5.1. Две стадии

| Стадия | Данные | Цель |
|--------|--------|------|
| **Pretrain** | 100K синтетических пар ДНК | Выучить базовую геометрию полосы |

### 5.2. Синтетические данные

Генерация без дорогого NW:
1. Случайная последовательность seq₁ длины L
2. Мутации (замены + инсерции + делеции) → seq₂
3. Путь выравнивания известен из процесса мутации
4. Вычисление (centre_diag, true_half_width) за O(n) по известному пути

**Стратификация по дивергенции:**
- Low: 0–10% мутаций
- Medium: 10–25%
- High: 25–50%

Равные доли + WeightedRandomSampler при обучении.

## 6. C++ модуль выравнивания

### 6.1. Компоненты

| Файл | Алгоритм | Сложность |
|------|----------|-----------|
| `full_nw.cpp` | Полный NW | O(n²) время, O(n²) память |
| `banded_nw.cpp` | Banded NW | O(n·W) время, O(n·W) память |
| `simd_banded_nw.cpp` | SIMD banded | O(n·W) с AVX2 векторизацией |
| `hirschberg.cpp` | Hirschberg (divide & conquer) | O(n·W) время, O(W) память |
| `four_russians.cpp` | Four Russians | O(n·W/log(t)) амортизированно |
| `band_doubling.cpp` | Orchestrator | Запускает banded → проверка escape → doubling |
| `profile_dp.cpp` | Профильное выравнивание | Для merge на внутренних узлах |

### 6.2. Band Doubling — гарантия корректности

```
Предсказание нейросети: (centre=5, W=20)
         │
         ▼
   Banded NW с W=20
         │
    Путь вышел ─── нет ──→ Готово! Результат корректен.
    за полосу?
         │
        да
         │
   ┌─────┴─────┐
   │ Детектируем │  escape_left? escape_right?
   │ направление │
   └─────┬─────┘
         │
   Удваиваем W (асимметрично)
   W_left × 2 или W_right × 2
         │
    Повторяем banded NW
         │
    ... (обычно 0–1 удвоений)
```

**Ключевой инвариант:** при W ≥ n/2 banded NW эквивалентен полному NW.
Значит band doubling **всегда** сходится к правильному ответу.

### 6.3. Four Russians ускорение

Идея: DP-матрица состоит из повторяющихся блоков t×t. Если блок с такими
входными значениями уже был вычислен — берём результат из lookup-таблицы.

```
hit_ratio = повторные_блоки / все_блоки
```

С ростом числа обработанных пар hit_ratio увеличивается (таблица заполняется).
Ожидаемый финальный hit_ratio: 30–70%.

## 7. Итеративное уточнение (Refinement)

Алгоритм в стиле MUSCLE:

```
Для каждого прохода (3 прохода):
    Для каждой seq_i (или 30% при N>100):
        1. Убрать seq_i из MSA
        2. Удалить пустые столбцы
        3. Построить профиль оставшихся
        4. Предсказать полосу: NN(seq_i, profile) → (centre, W)
        5. Выровнять seq_i к профилю
        6. Вставить обратно
        7. Если SP-score улучшился → принять, иначе → откатить
```

Ожидаемый прирост SP: +0.02–0.10.

## 9. Метрики качества MSA

### SP-score (Sum-of-Pairs)
Для каждой пары последовательностей (i, j) и каждой позиции в эталонном
выравнивании: проверяем, находятся ли те же два остатка в одном столбце
в предсказанном выравнивании.

```
SP = (число правильно совмещённых пар) / (общее число пар в эталоне)
```

Диапазон: 0.0 (случайное) – 1.0 (идеальное).

### TC-score (Total Column)
Доля столбцов эталона, которые полностью совпадают в предсказании
(все N остатков в одном столбце).

```
TC = (число полностью правильных столбцов) / (общее число столбцов с ≥2 остатками)
```

TC строже SP — один неправильный остаток обнуляет весь столбец.

## 10. Тестирование

### 10.1. Автоматические тесты (pytest)

| Тест | Что проверяет | Критерий |
|------|---------------|----------|
| `test_correctness` | banded + doubling ≡ full NW | Score совпадает с точностью 0.01 |
| `test_four_russians` | FR lookup + корректность | Таблица растёт, scores совпадают |
| `test_msa_quality` | BAliBASE SP/TC scores | SP > 0.1 для нашего метода |
| `test_speed_pairwise` | Ускорение vs full NW | Speedup ≥ 1.5× |
| `test_neural_vs_fixed` | Neural vs Fixed band | Score match rate > 80% |

### 10.2. Что гарантирует корректность

1. **Band doubling инвариант:** при любом предсказании нейросети результат
   выравнивания математически идентичен full NW. Это доказывается тем, что
   при W ≥ max_deviation удвоение прекращается, а при W = n/2 banded ≡ full.

2. **Тест test_correctness:** для 20+ пар разных длин и дивергенций
   проверяется, что score banded+doubling == score full NW.

3. **Hirschberg корректность:** divide-and-conquer не меняет score,
   только использует O(W) памяти вместо O(n·W).

### 10.3. Валидация на BAliBASE

BAliBASE 3.0 — стандартный бенчмарк для MSA:
- **RV11–RV30** — обучающие группы
- **RV40** — валидация
- **RV50** — тест
- Каждая группа: 3–50 белковых последовательностей + эталонное выравнивание

## 13. Новизна работы

1. **Нейросетевое предсказание band параметров** — вместо фиксированной
   ширины, модель адаптивно подбирает (centre, W) для каждой пары, учитывая
   длины, состав, дивергенцию.

2. **Гарантированная корректность** — band doubling математически гарантирует,
   что результат идентичен полному NW, независимо от качества предсказания.

3. **Гибридная архитектура** — CNN для пространственной структуры dot-plot +
   MLP для агрегированных k-mer признаков + embedding для типа последовательности.

4. **Асимметричная функция потерь** — штрафует недооценку ширины в 5× сильнее,
   что снижает число дорогих удвоений.

5. **Интеграция Four Russians + SIMD + Hirschberg** в единый C++ модуль
   для максимального ускорения внутри banded NW.
